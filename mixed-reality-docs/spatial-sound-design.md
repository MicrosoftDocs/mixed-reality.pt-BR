## <a name="four-key-things-spatial-sound-does-for-mixed-reality-development"></a><span data-ttu-id="dc04a-116">As quatro principais coisas que o som espacial faz para o desenvolvimento de realidade misturada</span><span class="sxs-lookup"><span data-stu-id="dc04a-116">Four key things spatial sound does for mixed reality development</span></span>

<span data-ttu-id="dc04a-117">Por padrão, os sons são reproduzidos no estéreo.</span><span class="sxs-lookup"><span data-stu-id="dc04a-117">By default, sounds are played back in stereo.</span></span> <span data-ttu-id="dc04a-118">Isso significa que o som será tocado sem nenhuma posição espacial, portanto, o usuário não sabe de onde veio o som.</span><span class="sxs-lookup"><span data-stu-id="dc04a-118">This means the sound will play with no spatial position, so the user does not know where the sound came from.</span></span> <span data-ttu-id="dc04a-119">O som espacial faz quatro coisas importantes para o desenvolvimento de realidade misturada:</span><span class="sxs-lookup"><span data-stu-id="dc04a-119">Spatial sound does four key things for mixed reality development:</span></span>

<span data-ttu-id="dc04a-120">**Aterramento**</span><span class="sxs-lookup"><span data-stu-id="dc04a-120">**Grounding**</span></span>

<span data-ttu-id="dc04a-121">Sem som, os objetos virtuais deixam de existir efetivamente quando viramos o nosso nosso rumo.</span><span class="sxs-lookup"><span data-stu-id="dc04a-121">Without sound, virtual objects effectively cease to exist when we turn our head away from them.</span></span> <span data-ttu-id="dc04a-122">Assim como os objetos reais, você deseja ser capaz de ouvir esses objetos mesmo quando não consegue vê-los e deseja localizá-los em qualquer lugar em sua parte.</span><span class="sxs-lookup"><span data-stu-id="dc04a-122">Just like real objects, you want to be able to hear these objects even when you can't see them, and you want to be able to locate them anywhere around you.</span></span> <span data-ttu-id="dc04a-123">Assim como os objetos virtuais precisam ser aterrados visualmente para misturar com o mundo real, eles também precisam ser forma audíveldos.</span><span class="sxs-lookup"><span data-stu-id="dc04a-123">Just as virtual objects need to be grounded visually to blend with your real world, they also need to be grounded audibly.</span></span> <span data-ttu-id="dc04a-124">O som espacial combina diretamente o seu ambiente de áudio real no mundo inteiro com o ambiente de áudio digital.</span><span class="sxs-lookup"><span data-stu-id="dc04a-124">Spatial sound seamlessly blends your real world audio environment with the digital audio environment.</span></span>

<span data-ttu-id="dc04a-125">**Atenção do usuário**</span><span class="sxs-lookup"><span data-stu-id="dc04a-125">**User attention**</span></span>

<span data-ttu-id="dc04a-126">Em experiências de realidade misturada, você não pode supor onde seu usuário está olhando e esperar que eles vejam algo que você coloca no mundo visualmente.</span><span class="sxs-lookup"><span data-stu-id="dc04a-126">In mixed reality experiences, you can't assume where your user is looking and expect them to see something you place in the world visually.</span></span> <span data-ttu-id="dc04a-127">Mas os usuários sempre podem ouvir uma reprodução de som mesmo quando o objeto que está tocando o som está por trás deles.</span><span class="sxs-lookup"><span data-stu-id="dc04a-127">But users can always hear a sound play even when the object playing the sound is behind them.</span></span> <span data-ttu-id="dc04a-128">As pessoas são usadas para ter sua atenção desenhada por som-nós instinctuallymos em direção a um objeto que ouvimos em nosso lugar.</span><span class="sxs-lookup"><span data-stu-id="dc04a-128">People are used to having their attention drawn by sound - we instinctually look toward an object that we hear around us.</span></span> <span data-ttu-id="dc04a-129">Quando você deseja direcionar o olhar do usuário para um local específico, em vez de usar uma seta para apontar visualmente, colocar um som nesse local é uma maneira muito natural e rápida de orientá-los.</span><span class="sxs-lookup"><span data-stu-id="dc04a-129">When you want to direct your user's gaze to a particular place, rather than using an arrow to point them visually, placing a sound in that location is a very natural and fast way to guide them.</span></span>

<span data-ttu-id="dc04a-130">**Imersão**</span><span class="sxs-lookup"><span data-stu-id="dc04a-130">**Immersion**</span></span>

<span data-ttu-id="dc04a-131">Quando os objetos se movem ou colidem, geralmente ouvimos essas interações entre os materiais.</span><span class="sxs-lookup"><span data-stu-id="dc04a-131">When objects move or collide, we usually hear those interactions between materials.</span></span> <span data-ttu-id="dc04a-132">Então, quando os objetos não fazem o mesmo som que fariam no mundo real, um nível de imersão é perdido, como assistir a um filme assustador com o volume em todo o caminho.</span><span class="sxs-lookup"><span data-stu-id="dc04a-132">So when your objects don't make the same sound they would in the real world, a level of immersion is lost - like watching a scary movie with the volume all the way down.</span></span> <span data-ttu-id="dc04a-133">Todos os sons do mundo real são provenientes de um ponto específico no espaço, quando viramos nossos cabeçotes, ouvimos a alteração de onde esses sons estão vindos em relação aos nossos ouvidos e podemos acompanhar o local de qualquer som dessa maneira.</span><span class="sxs-lookup"><span data-stu-id="dc04a-133">All sounds in the real world come from a particular point in space - when we turn our heads, we hear the change in where those sounds are coming from relative to our ears, and we can track the location of any sound this way.</span></span> <span data-ttu-id="dc04a-134">Os sons espaciais formam a "sensação" de um lugar além do que podemos ver.</span><span class="sxs-lookup"><span data-stu-id="dc04a-134">Spatialized sounds make up the "feel" of a place beyond what we can see.</span></span>

<span data-ttu-id="dc04a-135">**Design de interação**</span><span class="sxs-lookup"><span data-stu-id="dc04a-135">**Interaction design**</span></span>

<span data-ttu-id="dc04a-136">Na maioria das experiências interativas tradicionais, a interação parece que os efeitos de som da interface do usuário são reproduzidos em mono ou estéreo padrão.</span><span class="sxs-lookup"><span data-stu-id="dc04a-136">In most traditional interactive experiences, interaction sounds like UI sound effects are played in standard mono or stereo.</span></span> <span data-ttu-id="dc04a-137">Mas como tudo em realidade misturada existe no espaço 3D-incluindo a interface do usuário-esses objetos se beneficiam de sons espaciais.</span><span class="sxs-lookup"><span data-stu-id="dc04a-137">But because everything in mixed reality exists in 3D space - including the UI - these objects benefit from spatialized sounds.</span></span> <span data-ttu-id="dc04a-138">Quando pressionamos um botão no mundo real, o som que ouvimos é proveniente desse botão.</span><span class="sxs-lookup"><span data-stu-id="dc04a-138">When we press a button in the real world, the sound we hear comes from that button.</span></span> <span data-ttu-id="dc04a-139">Ao espacialar os sons de interação, fornecemos novamente uma experiência de usuário mais natural e realista.</span><span class="sxs-lookup"><span data-stu-id="dc04a-139">By spatializing interaction sounds, we again provide a more natural and realistic user experience.</span></span>

## <a name="best-practices-when-using-spatial-sound"></a><span data-ttu-id="dc04a-140">Práticas recomendadas ao usar som espacial</span><span class="sxs-lookup"><span data-stu-id="dc04a-140">Best practices when using spatial sound</span></span>

<span data-ttu-id="dc04a-141">**Sons reais funcionam melhor do que sons sintetizados ou não naturais**</span><span class="sxs-lookup"><span data-stu-id="dc04a-141">**Real sounds work better than synthesized or unnatural sounds**</span></span>

<span data-ttu-id="dc04a-142">Quanto mais familiar o usuário estiver com um tipo de som, mais real ele se sentirá e, mais facilmente, ele poderá localizá-lo em seu ambiente.</span><span class="sxs-lookup"><span data-stu-id="dc04a-142">The more familiar your user is with a type of sound, the more real it will feel, and the more easily they will be able to locate it in their environment.</span></span> <span data-ttu-id="dc04a-143">Uma voz humana, por exemplo, é um tipo muito comum de som, e os usuários vão localizá-lo tão rapidamente quanto uma pessoa na sala conversando com eles.</span><span class="sxs-lookup"><span data-stu-id="dc04a-143">A human voice, for example, is a very common type of sound, and your users will locate it just as quickly as a real person in the room talking to them.</span></span>

<span data-ttu-id="dc04a-144">**Simulação de trunfos de expectativa**</span><span class="sxs-lookup"><span data-stu-id="dc04a-144">**Expectation trumps simulation**</span></span>

<span data-ttu-id="dc04a-145">Se você for usado para um som proveniente de uma determinada direção, sua atenção será guiada nessa direção, independentemente das indicações espaciais. Por exemplo, na maioria das vezes que ouvimos pássaros, eles estão acima dos EUA.</span><span class="sxs-lookup"><span data-stu-id="dc04a-145">If you are used to a sound coming from a particular direction, your attention will be guided in that direction regardless of spatial cues. For example, most of the time that we hear birds, they are above us.</span></span> <span data-ttu-id="dc04a-146">Tocar o som de um pássaro provavelmente fará com que o usuário pesquise, mesmo que você coloque o som abaixo deles.</span><span class="sxs-lookup"><span data-stu-id="dc04a-146">Playing the sound of a bird will most likely cause your user to look up, even if you place the sound below them.</span></span> <span data-ttu-id="dc04a-147">Isso geralmente é confuso e é recomendável que você trabalhe com expectativas como essas em vez de ir contra elas para uma experiência mais natural.</span><span class="sxs-lookup"><span data-stu-id="dc04a-147">This is usually confusing, and it is recommended that you work with expectations like these rather than going against them for a more natural experience.</span></span>

<span data-ttu-id="dc04a-148">**A maioria dos sons deve ser espacial**</span><span class="sxs-lookup"><span data-stu-id="dc04a-148">**Most sounds should be spatialized**</span></span>

<span data-ttu-id="dc04a-149">Conforme mencionado acima, tudo em realidade misturada existe no espaço 3D-seus sons também devem ser.</span><span class="sxs-lookup"><span data-stu-id="dc04a-149">As mentioned above, everything in Mixed Reality exists in 3D space - your sounds should as well.</span></span> <span data-ttu-id="dc04a-150">Às vezes, as músicas podem se beneficiar da espacial, especialmente quando estão ligadas a um menu ou a alguma outra interface do usuário.</span><span class="sxs-lookup"><span data-stu-id="dc04a-150">Even music can sometimes benefit from spatialization, particularly when it's tied to a menu or some other UI.</span></span>

<span data-ttu-id="dc04a-151">**Evitar emissores invisíveis**</span><span class="sxs-lookup"><span data-stu-id="dc04a-151">**Avoid invisible emitters**</span></span>

<span data-ttu-id="dc04a-152">Como temos sido condicionados a olhar os sons que ouvimos em nosso lugar, pode ser uma experiência não natural e até mesmo unnerving para localizar um som que não tenha nenhuma presença Visual.</span><span class="sxs-lookup"><span data-stu-id="dc04a-152">Because we've been conditioned to look at sounds that we hear around us, it can be an unnatural and even unnerving experience to locate a sound that has no visual presence.</span></span> <span data-ttu-id="dc04a-153">Os sons do mundo real não vêm de espaço vazio, então certifique-se de que se um emissor de áudio for colocado dentro do ambiente imediato do usuário que ele também possa ser visto.</span><span class="sxs-lookup"><span data-stu-id="dc04a-153">Sounds in the real world don't come from empty space, so be sure that if an audio emitter is placed within the user's immediate environment that it can also be seen.</span></span>

<span data-ttu-id="dc04a-154">**Evitar mascaramento espacial**</span><span class="sxs-lookup"><span data-stu-id="dc04a-154">**Avoid spatial masking**</span></span>

<span data-ttu-id="dc04a-155">O som espacial se baseia em indicações acústicas muito sutis que podem ser sobreligadas por outros sons.</span><span class="sxs-lookup"><span data-stu-id="dc04a-155">Spatial sound relies on very subtle acoustic cues that can be overpowered by other sounds.</span></span> <span data-ttu-id="dc04a-156">Se você tiver música estéreo ou sons de ambiente, verifique se eles são baixos o suficiente na combinação para dar espaço para os detalhes de seus sons espaciais que permitirão que os usuários os localizem facilmente e os mantenham informais reais e naturais.</span><span class="sxs-lookup"><span data-stu-id="dc04a-156">If you do have stereo music or ambient sounds, make sure they are low enough in the mix to give room for the details of your spatialized sounds that will allow your users to locate them easily, and keep them sounding realistic and natural.</span></span>

## <a name="general-concepts-to-keep-in-mind-when-using-spatial-sound"></a><span data-ttu-id="dc04a-157">Conceitos gerais para ter em mente ao usar som espacial</span><span class="sxs-lookup"><span data-stu-id="dc04a-157">General concepts to keep in mind when using spatial sound</span></span>

<span data-ttu-id="dc04a-158">**O som espacial é uma simulação**</span><span class="sxs-lookup"><span data-stu-id="dc04a-158">**Spatial sound is a simulation**</span></span>

<span data-ttu-id="dc04a-159">O uso mais frequente de som espacial é fazer com que pareça que ele está emanando de um objeto real ou virtual no mundo.</span><span class="sxs-lookup"><span data-stu-id="dc04a-159">The most frequent use of spatial sound is making a sound seem as though it is emanating from a real or virtual object in the world.</span></span> <span data-ttu-id="dc04a-160">Assim, os sons espaciais podem fazer mais sentido provenientes desses objetos.</span><span class="sxs-lookup"><span data-stu-id="dc04a-160">Thus, spatialized sounds may make the most sense coming from such objects.</span></span>

<span data-ttu-id="dc04a-161">Observe que a precisão percebida do som espacial significa que um som não deve ser necessariamente emitido a partir do centro de um objeto, pois a diferença será perceptível, dependendo do tamanho do objeto e da distância do usuário.</span><span class="sxs-lookup"><span data-stu-id="dc04a-161">Note that the perceived accuracy of spatial sound means that a sound shouldn't necessarily emit from the center of an object, as the difference will be noticeable depending on the size of the object and distance from the user.</span></span> <span data-ttu-id="dc04a-162">Com objetos pequenos, o ponto central do objeto geralmente é suficiente.</span><span class="sxs-lookup"><span data-stu-id="dc04a-162">With small objects, the center point of the object is usually sufficient.</span></span> <span data-ttu-id="dc04a-163">Para objetos maiores, você pode querer um emissor de som ou vários emissores no local específico dentro do objeto que deve estar produzindo o som.</span><span class="sxs-lookup"><span data-stu-id="dc04a-163">For larger objects, you may want a sound emitter or multiple emitters at the specific location within the object that is supposed to be producing the sound.</span></span>

<span data-ttu-id="dc04a-164">**Normalizar todos os sons**</span><span class="sxs-lookup"><span data-stu-id="dc04a-164">**Normalize all sounds**</span></span>

<span data-ttu-id="dc04a-165">A atenuação de distância ocorre rapidamente no primeiro medidor do usuário, como faz no mundo real.</span><span class="sxs-lookup"><span data-stu-id="dc04a-165">Distance attenuation happens quickly within the first meter from the user, as it does in the real world.</span></span> <span data-ttu-id="dc04a-166">Todos os arquivos de áudio devem ser normalizados para garantir a atenuação de distância fisicamente precisa e garantir que um som possa ser ouvido quando houver vários medidores de distância (quando aplicável).</span><span class="sxs-lookup"><span data-stu-id="dc04a-166">All audio files should be normalized to ensure physically accurate distance attenuation, and ensure that a sound can be heard when several meters away (when applicable).</span></span> <span data-ttu-id="dc04a-167">O mecanismo de áudio espacial tratará a atenuação necessária para um som "sentir" como está em uma determinada distância (com uma combinação de atenuação e "indicações de distância") e a aplicação de qualquer atenuação sobre isso pode reduzir o efeito.</span><span class="sxs-lookup"><span data-stu-id="dc04a-167">The spatial audio engine will handle the attenuation necessary for a sound to "feel" like it's at a certain distance (with a combination of attenuation and "distance cues"), and applying any attenuation on top of that could reduce the effect.</span></span> <span data-ttu-id="dc04a-168">Fora da simulação de um objeto real, a distância inicial decaimento de sons de *som espaciais* provavelmente será mais do que suficiente para uma combinação adequada do seu áudio.</span><span class="sxs-lookup"><span data-stu-id="dc04a-168">Outside of simulating a real object, the initial distance decay of *spatial sound* sounds will likely be more than enough for a proper mix of your audio.</span></span>

<span data-ttu-id="dc04a-169">**Descoberta de objeto e interfaces de usuário**</span><span class="sxs-lookup"><span data-stu-id="dc04a-169">**Object discovery and user interfaces**</span></span>

<span data-ttu-id="dc04a-170">Ao usar as indicações de áudio para direcionar a atenção do usuário para além de sua exibição atual, o som deve ser audível e proeminente na mistura, bem acima de qualquer som estéreo, e quaisquer outros sons espaciais que possam atrapalhar a indicação de áudio direcional.</span><span class="sxs-lookup"><span data-stu-id="dc04a-170">When using audio cues to direct the user's attention beyond their current view, the sound should be audible and prominent in the mix, well above any stereo sounds, and any other spatialized sounds which might distract from the directional audio cue.</span></span> <span data-ttu-id="dc04a-171">Para sons e músicas associados a um elemento da interface do usuário (por exemplo, um menu), o emissor de som deve ser anexado a esse objeto.</span><span class="sxs-lookup"><span data-stu-id="dc04a-171">For sounds and music that are associated with an element of the user interface (e.g. a menu), the sound emitter should be attached to that object.</span></span> <span data-ttu-id="dc04a-172">O estéreo e outra reprodução de áudio não posicional podem dificultar a localização de elementos espaciais para os usuários (veja acima: Evite mascaramento espacial).</span><span class="sxs-lookup"><span data-stu-id="dc04a-172">Stereo and other non-positional audio playing can make spatialized elements difficult for users to locate (See above: Avoid spatial masking).</span></span>

<span data-ttu-id="dc04a-173">**Usar som espacial sobre som 3D padrão o máximo possível**</span><span class="sxs-lookup"><span data-stu-id="dc04a-173">**Use spatial sound over standard 3D sound as much as possible**</span></span>

<span data-ttu-id="dc04a-174">Em realidade misturada, para a melhor experiência do usuário, o áudio 3D deve ser obtido usando o som espacial em vez de tecnologias de áudio 3D herdadas.</span><span class="sxs-lookup"><span data-stu-id="dc04a-174">In mixed reality, for the best user experience, 3D audio should be achieved using spatial sound rather than legacy 3D audio technologies.</span></span> <span data-ttu-id="dc04a-175">Em geral, a espacial aprimorada vale a pena ter um pequeno custo de CPU em relação ao som 3D padrão.</span><span class="sxs-lookup"><span data-stu-id="dc04a-175">In general, the improved spatialization is worth the small CPU cost over standard 3D sound.</span></span> <span data-ttu-id="dc04a-176">O áudio 3D padrão pode ser usado para sons de baixa prioridade, sons que são espaciais, mas não necessariamente vinculados a um objeto físico ou virtual, e os objetos que o usuário nunca precisam localizar para interagir com o aplicativo.</span><span class="sxs-lookup"><span data-stu-id="dc04a-176">Standard 3D audio can be used for low-priority sounds, sounds that are spatialized but not necessarily tied to a physical or virtual object, and objects that the user never need locate to interact with the app.</span></span>
